# Natural-Language-Processing-NLP-_Application_for_Country_Name_Analysis_Using_World_Bank_Data
This project is a Natural Language Processing (NLP) application built in Python to analyze textual patterns in country and region names from a cleaned World Bank country statistics dataset. The project combines text preprocessing, tokenization, frequency analysis, n-gram extraction, part-of-speech tagging, and statistical correlation to uncover meaningful linguistic and structural patterns in geographic names. By integrating NLP techniques with structured socioeconomic data, the project demonstrates how text analytics can be applied beyond traditional sentiment analysis or document classification and used instead for exploratory analysis of real-world datasets.

The main objective of this project is to explore whether country and region names contain hidden linguistic patterns that may relate to geographic or demographic variables such as population density, while also identifying the most common words, word pairs, and grammatical structures found in these names. This makes the project both a practical NLP exercise and an interesting example of how language patterns can be studied in structured datasets.

The project begins by importing the required Python libraries, including Pandas for data handling and NLTK (Natural Language Toolkit) for natural language processing tasks. Several NLTK resources are downloaded, such as tokenizers, stopword lists, and part-of-speech taggers, to ensure the notebook can process and analyze text properly. The dataset used in the project is a cleaned CSV file named cleaned_worldbank_country_stats.csv, which contains country-level and region-level records along with variables such as country names, density values, and an indicator showing whether an entry is a country or a region. The data is loaded into a DataFrame, and a copy of the cleaned dataset is preserved for analysis.

A custom tokenization function is then created to process the country column. This function converts each name to lowercase and tokenizes it into individual words using NLTK’s word_tokenize. For example, names like “United Arab Emirates” are split into separate tokens such as ["united", "arab", "emirates"]. This allows the project to transform country and region names from plain text into analyzable units. After tokenization, the project creates two new columns: one containing the list of tokens and another storing the word count for each country or region name. This step is essential because it provides a numerical representation of text complexity and supports later statistical analysis.

One of the most interesting parts of the project is its hypothesis-driven analysis. Rather than simply describing the text, the notebook structures its investigation into multiple hypotheses (H1–H5), each exploring a different linguistic pattern. The first hypothesis examines whether there is any relationship between the number of words in a country’s name and its population density. To do this, the project filters the dataset to include only actual countries (excluding regions) and calculates the correlation between word_count and density. This provides a simple but meaningful example of combining NLP-derived features with numeric socioeconomic variables. The interpretation in the notebook suggests that the correlation is weak, indicating that name length alone is not a strong predictor of population density, though there may be slight tendencies worth noting.

The second hypothesis focuses on word frequency analysis, which is a foundational NLP task. Before calculating word frequencies, the project removes common English stopwords such as “the,” “of,” and “and” so that the analysis emphasizes meaningful geographic and political terms instead of filler words. All tokens from the dataset are combined into a single list, and NLTK’s FreqDist is used to identify the most frequent words. The project then extracts the top five most common words across country and region names. This helps reveal recurring naming conventions, such as the frequent use of terms like “republic,” “islands,” or similar descriptors that reflect geography, governance, or colonial naming patterns.

The third hypothesis extends this idea by performing bigram analysis. Instead of looking at single words, the notebook generates two-word combinations (bigrams) from the tokenized country names, again excluding stopwords. This allows the analysis to identify common phrases such as “united arab” or “south africa,” which carry more semantic meaning than individual tokens alone. By counting the most frequent bigrams, the project captures patterns that are especially useful in place names, where meaning often depends on word combinations rather than isolated terms. This demonstrates an important NLP concept: context matters, and n-grams can provide richer insights than single-word frequency counts.

The fourth hypothesis applies part-of-speech (POS) tagging to understand the grammatical structure of country and region names. Each tokenized name is passed through NLTK’s pos_tag function, which assigns grammatical labels such as NN (noun), NNP (proper noun), and JJ (adjective). The notebook then counts how often each tag appears, excluding stopwords. The results show that nouns and proper nouns dominate the dataset, which is expected for geographic names, while adjectives also appear in multi-word names that describe political or geographic characteristics. This part of the project is valuable because it demonstrates how even short text strings like country names can be studied linguistically and categorized by grammatical composition.

The fifth and final hypothesis compares the average word count between countries and regions. Using the is_region indicator, the project calculates the average number of words in each group and compares them directly. This allows the project to test whether region names tend to be more complex or descriptive than country names. According to the notebook’s interpretation, regions often have a higher average word count, suggesting that they may use broader descriptive naming conventions than individual countries. This adds a comparative dimension to the project and reinforces the idea that text structure can reveal useful patterns in structured data.

All analysis results are organized into a Python dictionary, converted into a Pandas DataFrame, and displayed in a clean tabular format. This results table includes three components for each hypothesis: the analysis performed, the result, and a short interpretation. Finally, the results are exported to a CSV file named nlp_analysis_results.csv, making the project reproducible and allowing the findings to be reused in reports, dashboards, or future analyses.

Overall, this project is a strong example of how Natural Language Processing can be applied to structured datasets in a creative and analytical way. It demonstrates practical NLP techniques such as tokenization, stopword filtering, frequency distribution, n-gram extraction, and POS tagging, while also showing how textual features can be connected to quantitative variables like population density. The project is especially useful for students and aspiring data scientists because it highlights both the technical workflow of NLP and the importance of designing meaningful hypotheses. It can serve as a portfolio project for data science, NLP, exploratory data analysis, or applied machine learning and shows an ability to combine text analysis with domain-specific reasoning.
